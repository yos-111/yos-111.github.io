ADMINISTRAR
SERVIDOR WEB

La administración de un servidor web es el conjunto de tareas para mantener el software operativo, seguro y rápido .Se resume en tres acciones clave:

Configuración: Instalar el programa (Nginx, Apache) y enlazar los dominios de las páginas web.
Seguridad: Implementar certificados SSL (HTTPS), actualizar el sistema y programar respaldos.
Monitoreo: Supervisar el consumo de recursos (RAM/CPU) para evitar caídas del servicio.

4.1 cPANEL DE SERVIDOR WEB


cPanel es un panel de control gráfico basado en web que simplifica drásticamente la administración de servidores Linux y cuentas de hosting. Permite automatizar tareas complejas sin necesidad de dominar la consola de comandos. A través de su interfaz intuitiva dividida en bloques se pueden gestionar correos electrónicos corporativos, copias de seguridad automáticas, zonas DNS, bases de datos y la instalación de aplicaciones web (como WordPress) con un solo clic.



4.2 ADMINISTRADOR DE ARCHIVOS


Herramienta web integrada dentro de cPanel que funciona como el explorador de archivos del servidor.

Directorio Raíz Público: Toda la información que debe ser visible en internet tiene que alojarse obligatoriamente dentro del directorio llamado public_html (o www). Los archivos fuera de esta carpeta quedan ocultos al público por seguridad.
Funciones integradas: Permite crear, renombrar, mover, borrar y editar archivos directamente en el navegador de forma visual. También incluye herramientas para comprimir y descomprimir archivos planos (como archivos .zip), agilizando la subida masiva de código fuente.



4.3 PROTOCOLO DE TRANSFERENCIA DE ARCHIVOS (FTP)

Protocolo de red estándar (puerto 21) utilizado para transferir archivos entre un cliente local y el servidor remoto.

Estructura: Requiere un Servidor FTP corriendo en el hosting y un Cliente FTP local en la computadora del desarrollador (el software más popular y de código abierto es FileZilla).
Variantes de Seguridad: El protocolo FTP plano transmite las contraseñas y datos en texto sin formato, exponiéndolos a intercepciones. Por ello, se deben implementar sus variantes cifradas:
FTPS (FTP sobre SSL/TLS) o SFTP (SSH File Transfer Protocol, que viaja de forma segura cifrado a través del puerto 22 de SSH).
